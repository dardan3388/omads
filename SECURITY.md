# Security Policy

Thank you for helping keep OMADS safe.

## Supported Versions

OMADS is currently maintained on the active `main` branch and the latest release line.

| Version | Supported |
| --- | --- |
| `main` | Yes |
| Latest tagged release | Yes |
| Older releases | No |

## Reporting A Vulnerability

Please do **not** open a public GitHub issue for security vulnerabilities.

Instead, report security issues privately by email:

- `ceo@shala-mail.de`

Please include as much detail as possible:

- affected version, branch, or commit
- steps to reproduce
- expected impact
- any proof of concept, logs, or screenshots that help explain the issue

If the report is valid, OMADS will aim to acknowledge it within 5 business days and then follow up with the next steps.

## Disclosure Expectations

- Please give us a reasonable amount of time to investigate and prepare a fix before public disclosure.
- We will coordinate with the reporter when a fix, workaround, or advisory is ready.
- If a report turns out not to be a security issue, we may redirect it to the normal issue tracker.
